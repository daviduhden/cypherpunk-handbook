How to contribute

  1. Fork the repository on GitHub.
  2. Create a dedicated branch for your changes.
  3. Submit a pull request with a clear and descriptive summary.

Content guidelines

  • Keep content concise, precise, and actionable.
  • Use a respectful, formal, and neutral tone.
  • Avoid commercial endorsements or promotional language.
  • Write in English or Spanish; include translations whenever possible.
  • Use the repository template when available (see articles/).
  • Cite reliable sources and include links for tools, commands, and references.
  • Do not include personal data, credentials, or sensitive information.
  • Follow existing markup and formatting conventions (headings, paragraphs, <time> tags).
  • Include minimal metadata: author, tags, and a <time> element.
  • Optimize all media assets (images, audio, video, documents) and store them within the repository (e.g. images/).

Licensing

All contributions must comply with the project license located at ./license.html. If any content (code, images, or other assets) is not covered by this license, clearly declare the exception at the beginning of the file and include the applicable license or permission notice.

Suggested exception notice format (place near the top of the file):

EXCEPTIONS:
files/images/thirdparty.jpg — licensed under CC BY 4.0 (see LICENSE in images/)
some-code/snippet.c — used with permission from upstream (MIT)

Submitting articles

To publish a new article, create an HTML file under articles/. Include a header containing a <time> element that specifies the publication date.

Example:

<header>
<h1>Title</h1>
<time datetime="2025-12-18">18 Dec 2025</time>
</header>

Use the provided template for new articles and update the following files so the website and RSS feed reflect your changes:

data/articles.json
feeds/articles.xml

Quick links: data/articles.json — feeds/articles.xml

Repository & commands

Repository: https://github.com/daviduhden/cypherpunk-handbook

Recommended workflow using the GitHub CLI (gh):

gh repo fork daviduhden/cypherpunk-handbook --clone=true
cd cypherpunk-handbook
git checkout -b my-article
# Add your article under articles/
git add articles/my-article.html data/articles.json
git commit -m "Add article: My Article"
git push --set-upstream origin my-article
gh pr create --fill

Alternative workflow using plain Git:

git clone https://github.com/daviduhden/cypherpunk-handbook.git
cd cypherpunk-handbook
git checkout -b my-article
# Add and commit changes
git add articles/my-article.html data/articles.json
git commit -m "Add article: My Article"
git push origin my-article

Patches

Git patches may also be sent to the SimpleX group. Files must be attached uncompressed; archive formats (zip, tar, tar.gz, etc.) will be ignored. Prefer standard patch formats such as those generated by git format-patch.

Automation

Perl scripts located in scripts/ can be used to update and rebuild the site. These scripts regenerate index.html, data/articles.json, and feeds/articles.xml.

perl ./scripts/update-guides.pl
perl ./scripts/update-feeds.pl articles/my-article.html
perl ./scripts/rebuild-feeds.pl

Feeds

An RSS feed for articles is available at ./feeds/articles.xml.
